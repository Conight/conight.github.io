Hugo 与 PWA

2020-01-16 [Updated: 2020-01-16] :: Conight
#Hugo  #PWA  #WEB 

Progressive Web App (PWA)1 作为 Google 在 2016 年提出的概念，目的是让网页体验更加接近于原生应用。

本文记录把 PWA 结合 Hugo2 的过程，使个人博客更加现代化。

Service Workers⌗

注册⌗

首先对 Hugo 添加 Service Workers 支持的 Js 文件：sw.js。 文件可从 offline-first-sw 下载。

文件下载之后，放置到 Hugo 可读取的目录下。比如 /static 目录。

创建一个 footer 的 Partial Templates 并添加注册 Service Workers 的代码。

if('serviceWorker' in navigator) {
    const PREFETCH = true;
    const PREFETCH_LINK_RELS = ['index','next', 'prev', 'prefetch'];
    function prefetchCache() {
        if(navigator.serviceWorker.controller) {
            let links = document.querySelectorAll(
                PREFETCH_LINK_RELS.map((rel) => {
                    return 'link[rel='+rel+']';
                }).join(',')
            );
            if(links.length > 0) {
                Array.from(links)
                    .map((link) => {
                        let href = link.getAttribute('href');
                        navigator.serviceWorker.controller.postMessage({
                            action : 'cache',
                            url : href,
                        });
                    });
            }
        }
    }

    navigator.serviceWorker
        .register('/sw.js', { scope: '/' })
        .then(() => {
            console.log('Service Worker Registered');
        });

    navigator.serviceWorker
        .ready
        .then(() => {
            if(PREFETCH) {
                prefetchCache();
            }
        });
}

配置⌗

CACHE_VERSION { number }⌗

声明当前 Web APP 的版本，需要根据缓存的文件目录来决定是否更新版本号。 只有版本号变更，浏览器才会缓存新的资源文件并替换旧版本下的缓存。

const CACHE_VERSION = 1;

BASE_CACHE_FILES { array }⌗

定义始终需要缓存的文件、页面路径。

const BASE_CACHE_FILES = [
    // page cache
    '/',
    '/about/',
    '/zh/about/',
    '/showcase/',
    '/zh/showcase/',
    // assets cache
    '/assets/main.js',
    '/assets/prism.js',
    '/assets/style.css',
    '/manifest.json',
    // ico cache
    '/favicon.ico',
];

OFFLINE_CACHE_FILES { array }⌗

定义离线页面所需的文件。

const OFFLINE_CACHE_FILES = [
    '/offline.html',
];

NOT_FOUND_CACHE_FILES { array }⌗

定义 404 页面所需的文件。

const NOT_FOUND_CACHE_FILES = [
    '/404.html',
];

CACHE_BLACKLIST { array}⌗

资源缓存黑名单，在这个名单中的地址、域名不会缓存资源。建议不要包含本地测试地址。

const CACHE_BLACKLIST = [
    (str) => {
        return !str.startsWith('https://conight.com');
        // return !str.startsWith('http://localhost') && !str.startsWith('https://conight.com');
    },
];

manifest.json⌗

manifest 可以理解为 PWA 对于浏览器的声明文件。告诉浏览器该怎么安装你的 PWA。 具体字段定义可参考 官方文档 。

下面是我博客的 manifest：

{
  "name": "Conight's Blog",
  "short_name": "Conight's Blog",
  "description": "Share the world.",
  "icons": [
    {
      "src": "/android-chrome-192x192.png",
      "sizes": "192x192",
      "type": "image/png"
    },
    {
      "src": "/android-chrome-512x512.png",
      "sizes": "512x512",
      "type": "image/png"
    }
  ],
  "start_url": "/",
  "display": "standalone",
  "orientation": "portrait",
  "background_color": "#222128",
  "theme_color": "#222128"
}
  • 必须有 name 或者 short_name 其一。
  • 声明图标。也就是 icons 域。Google 官方建议 provide icons in increments of 48dp 。 可以使用在线工具 favicomatic.com 或 realfavicongenerator.net

放置 manifest 文件到网站目录并包含到 head 模板中。

<link rel="manifest" href="/manifest.json">
  • realfavicongenerator.net 网站生成的全套图标及资源文件中包含 manifest ，注意更改。

Lighthouse⌗

使用 Chrome 自带的检测工具测试下网站对 PWA 的支持。

Lighthouse, an open-source, automated tool for improving the quality of your Progressive Web Apps, eliminates much of the manual testing that was previously required. You can even use Lighthouse in continuous integration systems to catch regressions.

进入 Chrome 开发者模式，选择 Audits 功能，注意勾选 Audits -> Progressive Web Apps。 检测完成后会有支持情况的一个清单。

Lighthouse PWA Audits

所有的配置完成后，所有支持 PWA 的浏览器就会显示可安装的按钮或提示。

Install Notice

如果没有提示或显示可安装，可进入 Chrome 开发者模式下 Application -> Manifest， 其中 Installability 下则会提示可能的无法安装的原因：

Manifest Installability
